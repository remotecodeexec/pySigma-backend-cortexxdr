## Side Notes & Limitations
- Backend uses XQL syntax
- Pipeline uses XQL field names
- Pipeline supports `linux`, `windows`, and `macos` product types
- Pipeline supports the following category types for field mappings
  - `process_creation`
  - `file_event`
  - `file_change`
  - `file_rename`
  - `file_delete`
  - `image_load`
  - `registry_add`
  - `registry_delete`
  - `registry_event`
  - `registry_set`
  - `network_connection`
  - `firewall`
- Pipeline supports the following fields:
  - `CommandLine`
  - `Company`
  - `CurrentDirectory`
  - `DestinationHostname`
  - `DestinationIp`
  - `DestinationPort`
  - `Details`
  - `Image`
  - `ImageLoaded`
  - `IntegrityLevel`
  - `ParentCommandLine`
  - `ParentImage`
  - `ParentIntegrityLevel`
  - `ParentProcessId`
  - `ParentUser`
  - `ProcessId`
  - `Product`
  - `Protocol`
  - `SourceFilename`
  - `SourceHostname`
  - `SourceImage`
  - `SourceIp`
  - `SourcePort`
  - `TargetFilename`
  - `TargetObject`
  - `User`
  - `dst_ip`
  - `dst_port`
  - `md5`
  - `sha256`
  - `src_ip`
  - `src_port`
- Any unsupported fields or categories will throw errors
